# 9. 실행 루프: 계획, 수행, 검증, 기록

자연어 하네스의 실행 루프는 단순하다.

1. 읽기
2. 질문하기
3. 계획하기
4. 수행하기
5. 검증하기
6. 기록하기

중요한 것은 순서를 지키는 것이다. AI 세션은 급하게 수행 단계로 넘어갈수록 나중에 검토 비용이 커진다.

## 읽기

에이전트에게 먼저 읽을 문서를 지정한다.

```text
먼저 컨텍스트 패킷의 "반드시 읽기" 문서를 읽고,
이번 작업에 직접 영향을 주는 제약만 요약해줘.
아직 변경이나 작성은 하지 말아줘.
```

이 단계에서 사람이 요약을 보고 맥락 누락을 발견할 수 있다.

## 질문하기

모호한 점이 있으면 질문을 요구한다.

```text
작업을 진행하기 전에 완료 조건을 흔들 수 있는 질문만 해줘.
취향이나 사소한 구현 선택은 질문하지 말고 합리적으로 가정해도 된다.
```

질문도 하네스의 일부다. 질문이 너무 많으면 진행이 느려지고, 너무 적으면 추측이 늘어난다.

## 계획하기

계획은 길지 않아야 한다. 로컬 에이전트 세션에서는 짧고 검토 가능한 계획이 좋다.

```text
실행 계획을 5단계 이내로 제시해줘.
각 단계마다 검증 방법을 함께 적어줘.
```

계획 단계에서 범위가 커지면 즉시 줄인다.

## 수행하기

수행 단계에서는 한 번에 너무 많은 일을 맡기지 않는다. 작업을 작게 나눈다.

- 문서 초안 작성
- 코드 변경 방향 제안
- 리뷰 체크리스트 생성
- 테스트 케이스 후보 작성
- 릴리즈 노트 정리

자연어만 쓰는 하네스라도, 실제 프로젝트에서 에이전트가 코드를 수정할 수는 있다. 다만 이 책의 하네스 자체는 코드가 아니라 작업 통제 문서라는 점이 다르다.

## 검증하기

검증은 작업 헌장의 완료 조건과 평가 루브릭을 기준으로 한다.

```text
완료 조건별로 충족 여부를 표로 정리해줘.
증거가 없는 항목은 "미확인"으로 표시하고, 추측으로 통과 처리하지 마.
```

## 기록하기

마지막에 세션 장부를 남긴다.

```text
SESSION_LOG 항목을 작성해줘.
포함할 것:
- 목표
- 읽은 문서
- 중요한 결정
- 검증 결과
- 남은 위험
- 다음 세션에 전달할 맥락
- 하네스 문서에 반영할 개선점
```

기록은 귀찮지만 자연어 하네스를 팀 자산으로 만드는 핵심이다.
