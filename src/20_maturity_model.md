# 20. 성숙도 모델과 다음 단계

자연어 하네스는 한 번에 완성되지 않는다. 팀은 보통 개인 프롬프트에서 시작해, 프로젝트 지식베이스, 팀 운영 절차, 부분 자동화로 발전한다. 이 장은 현재 위치를 진단하고 다음 단계를 고르는 기준을 제공한다.

## Level 0: 개인 프롬프트

특징:

- 팀원마다 각자 프롬프트를 쓴다.
- 프로젝트 맥락은 매번 대화로 설명한다.
- 결과 검증은 개인 경험에 의존한다.
- 좋은 프롬프트가 있어도 공유되지 않는다.

대표 증상:

- “이거 전에 설명했는데”가 반복된다.
- AI 결과물의 품질이 사용자마다 크게 다르다.
- 실패 원인이 저장되지 않는다.

다음 단계:

- `PROJECT_CONTEXT.md` 한 페이지를 만든다.
- 공통 세션 시작 프롬프트를 만든다.

## Level 1: 최소 프로젝트 하네스

특징:

- `ai-harness/` 폴더가 있다.
- 프로젝트 컨텍스트와 평가 루브릭이 있다.
- 작업 헌장과 세션 로그를 일부 사용한다.

대표 증상:

- 문서는 있지만 최신성이 흔들린다.
- 팀원 일부만 사용한다.
- 템플릿이 너무 길거나 추상적이다.

다음 단계:

- 주간 하네스 리뷰를 시작한다.
- 회귀 노트를 실제 실패 사례에서 만들기 시작한다.

## Level 2: 팀 운영 하네스

특징:

- 작업 유형별 워크플로우가 있다.
- 평가 루브릭과 회귀 체크가 실제 리뷰에 쓰인다.
- 하네스 문서 변경에 소유자와 리뷰 절차가 있다.
- Claude, Codex, IDE Chat 등 도구별 세션 규칙이 있다.

대표 증상:

- 신규 팀원이 AI 세션을 더 빨리 시작한다.
- 반복 질문이 줄어든다.
- AI가 만든 결과물의 리뷰 코멘트가 더 구체적이다.

다음 단계:

- 세션 로그에서 반복 개선점을 추출한다.
- 안정된 절차를 일부 자동화한다.

## Level 3: 관찰 가능한 하네스

특징:

- 세션 로그, 평가 결과, 회귀 체크가 축적된다.
- 어떤 하네스 변경이 AI 행동을 개선했는지 추적한다.
- 실패 유형을 Context Gap, Ambiguity Gap, Evaluation Gap 등으로 분류한다.
- 오래된 문서를 폐기하는 절차가 작동한다.

대표 증상:

- 하네스 변경이 감이 아니라 근거로 이루어진다.
- 프롬프트 라이브러리에 검증 이력이 붙는다.
- 프로젝트별 하네스를 팀 템플릿으로 역수출할 수 있다.

다음 단계:

- 반복 절차를 도구화한다.
- 문서 최신성, 회귀 체크, 평가 결과를 자동 점검한다.

## Level 4: 부분 자동화 하네스

특징:

- 작업 헌장 생성, 세션 로그 수집, 회귀 체크 추천이 자동화된다.
- PR이나 이슈에 하네스 체크리스트가 연결된다.
- AI 세션 결과가 지식베이스 개선 제안으로 이어진다.

주의:

자동화는 성숙한 자연어 절차 위에 올려야 한다. 합의되지 않은 절차를 자동화하면, 팀은 왜 그런 규칙이 있는지 모른 채 도구에 끌려간다.

## 자가 진단 체크

```text
우리 팀의 현재 수준:

- [ ] AI 세션 시작 문서가 있다.
- [ ] 프로젝트 컨텍스트가 최신 상태다.
- [ ] 작업 헌장 없이 위험한 작업을 시작하지 않는다.
- [ ] 평가 루브릭으로 결과를 판정한다.
- [ ] 회귀 노트가 실제 실패에서 만들어진다.
- [ ] 하네스 문서 변경 이력이 있다.
- [ ] 팀원이 하네스 리뷰를 정기적으로 한다.
- [ ] 오래된 문서가 deprecated 처리된다.
- [ ] 안정된 절차 일부가 자동화되어 있다.
```

체크 수가 적다고 나쁜 팀은 아니다. 중요한 것은 다음 한 단계를 고르는 것이다.

## 다음 30일 계획

```text
1주차:
- ai-harness 최소 구조 생성
- PROJECT_CONTEXT 1페이지 작성
- 세션 시작 프롬프트 합의

2주차:
- 작업 헌장 2개 작성
- 세션 로그 2개 저장
- 실패 1개를 REGRESSION_NOTES에 기록

3주차:
- 평가 루브릭을 실제 AI 결과물 2개에 적용
- 프롬프트 라이브러리에 검증된 프롬프트 1개 추가
- 낡은 문서 1개 deprecated 처리

4주차:
- 주간 하네스 리뷰 진행
- 하네스 변경 PR 1개 작성
- 다음 달 자동화 후보 1개 선정
```

이 계획의 목표는 자연어 하네스를 “문서 프로젝트”가 아니라 “AI와 일하는 팀 운영 방식”으로 정착시키는 것이다.
